刷新页面后无法录音

Aug 6, 2021
1 min read

场景：在标注员录音任务中，当刷新页面，准备重新进行录音任务时，无法正常进行录音。

  • 原因：浏览器在授予用户录音权限前，需要用户有个交互操作，即主动设置获取录音权限。

解决方案1：不适用于最新版chrome浏览器

  • 在chrome浏览器中地址栏中输入chrome://flags/#autoplay-policy;

解决方案二

  • 设置一个获取录音权限的按钮，用户与其交互，主动设置获取录音权限。

对于方案二，又分为以下两种方案

  • 首先，在开发过程中，这两种方案有一点很难处理，一个特判的地方：假如之前标注员对第一个录音标注任务已经录过音，那么如果不做处理—方案2.1：第一个录音是会被展现出来，但录音的时候没有波形；不更改watch选项的方案2.2：第一个录音不会被展现出来，但是录音的时候会有波形。

  • 不更改watch选项的方案2.2：刷新页面，点击录音权限之后，就会出现以下情况特例。此时可以录音，且有波形，但是之前已经录过的音不会被展现出来，仅只有第一条会被隐藏。点击下一题，再点击上一题，则会被展现。两种方案都有这个情况。

  • 正确情况如下

  • initWave — 初始化波形
initWave() {
  this.wavesurfer = WaveSurfer.create({
    container: document.querySelector('#wave'),
    backend: 'MediaElement',
    waveColor: '#A8DBA8',
    progressColor: '#3B8686',
    mediaControls: false,
    plugins: [
      Timeline.create({
        container: document.querySelector('#wave-timeline')
      })
    ]
  });
  this.wavesurferMP = WaveSurfer.create({
    container: '#micorPhone',
    waveColor: 'black',
    interact: false,
    cursorWidth: 0,
    plugins: [
      Microphone.create({})
    ]
  });
}
  • initRecord — 获取录音对象
initRecord() {
  try {
    // webkit shim
    window.AudioContext = window.AudioContext || window.webkitAudioContext;
    navigator.getUserMedia = navigator.getUserMedia || navigator.webkitGetUserMedia;
    window.URL = window.URL || window.webkitURL;
    
    window.audio_context = window.audio_context || new AudioContext;
    console.log('录音权限请求');
  } catch (e) {
    console.log('进入catch3');
    this.$mtd.message.error('浏览器不支持录音');
  }

  navigator.getUserMedia({audio: true}, this.startUserMedia, function(e) {
    this.$mtd.message.error('没有可用输入' + e);
  });
},
startUserMedia(stream) {
  var input = window.audio_context.createMediaStreamSource(stream);
  this.$mtd.message.success('媒体流被创建');
  
  window.recorder = new Recorder(input);
  this.$mtd.message.success('录音对象被创建');
  this.recordInited = true
}

方案2.1

  • 优点：可以解决刷新页面造成的任务不能录音的问题 + 不会存在上述特例情况。

  • 缺点：录音的时候没有波纹振动表现 。

  1. 先在mounted生命周期钩子函数中先初始化波形。
  2. 再在获取录音权限绑定的方法里初始化录音对象，并主动设置获取录音权限
  • mounted — 模板从内存中被渲染视图触发的生命周期钩子函数
mounted() {
  this.initWave(); //初始化波形方法
}
  • getRecordPermission — 获取录音权限按钮绑定的方法
// 按钮点击触发的方法
getRecordPermission() {
  this.initRecord();
  window.audio_context.resume();
  this.permission = true;
  this.$mtd.message.success('获取录音权限成功');
}
  • vue watch选项
watch: {
  // wqt-start
  content() {
    this.loadWave(this.content);
  },
  // wqt-end
  config() {
    this.initAnswers();
  }
}

方案2.2

  • 如上特例的解决方案：第一次刷新的时候不走监听content的逻辑，走监听permission（是否获取到录音权限的标记）的逻辑。

  • 优点：方案2.1的优点 + 录音的时候有波纹振动表现.

  • 缺点：使用watch监听的属性content（录音链接，之前的录音），首页刷新的时候若第一个录音标注任务已经录过音，那么它的content会变一次，触发底层watcher。所以，他会进入监听逻辑，但第一次什么都没有做。

  • getRecordPermission — 获取录音权限按钮绑定的方法

getRecordPermission() {
  this.initRecord();
  window.audio_context.resume();
  this.initWave();
  this.permission = true;
  this.$mtd.message.success('获取录音权限成功');
}
  • vue watch选项
watch: {
  // wqt-start
  content() {
    if (this.permission) {
      this.loadWave(this.content);
    }
  },
  permission() {
    this.loadWave(this.content);
  },
  // wqt-end
  config() {
    this.initAnswers();
  }
}
