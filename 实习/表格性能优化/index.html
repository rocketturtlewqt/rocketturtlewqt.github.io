表格性能优化

Jul 25, 2021
1 min read

  • <任务完成时间>是单独的一个db接口，抽取数据结果的时间略长。同时，<平均每道题目完成时间>依赖于<任务完成时间>和完成题目个数。
  • 其它的列是单独的一个接口，当使用这两个接口同时生成表格时，换页会有明显较长时间的卡顿。

解决方案1

  • 将两个接口分流，先用时间较短的接口进行表格框架的生成，再用db操作时间较长的接口填充剩余表格。

  • 将针对任务完成时间接口单独抽离出来，用异步请求的方式将结果集包装成一个数组，一次性渲染。

  • 缺陷：表格框架生成较快，但是任务时间列表可能还是需要等待一段时间才能被渲染。

解决方案2

  • 将两个接口分流，先用时间较短的接口进行表格框架的生成，再用db操作时间较长的接口填充剩余表格。

  • 与方案1不同的是，针对任务完成时间，即时请求即时渲染，不需要包装成结果集。

  • 缺陷：渲染顺序比较乱。

async getFinishTime() {
  const taskList = this.tableData;
  taskList.forEach((task, index) => {
    const rel = this.ajax.post('/annotation/api/getTaskListTime', { taskId: task.id });
    const option = this.tableData[index];
    rel.then(val => {
      if (val && val.data && val.data.result) {
        const finishTime = val.data.result.tagger_time_all;
        if (finishTime) {
          option.finish_time = this.changeTimeObjToString(this.changeTime(finishTime));
        } else {
          option.finish_time = '未开始答题';
        }
        const num = option.total_count - option.surplus_count;
        const averageSeconds = Math.floor(finishTime / num);
        if (option.finish_time === '未开始答题') {
          option.average = '未开始答题';
        } else if (option.total_count - option.surplus_count === 0) {
          option.average = '无穷大';
        } else {
          option.average = this.changeTimeObjToString(this.changeTime(averageSeconds));
        }
      } else {
        throw new Error('调用getTaskListTime接口的path路径存在undefined');
      }
    });
  });
}
