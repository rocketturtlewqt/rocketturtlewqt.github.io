上传全局配置文件视图渲染复杂度优化

Aug 4, 2021
1 min read

上传全局配置文件

  • 上传全局配置文件成功后，因前端不会再次发起请求，需要做个渲染处理。

方案一

  • 将配置文件中的机器人配置读取出来，扔进数组里按照场景号排序，复杂度(nlogn)

方案二

  • 观察到原数组是有序的，可以从后向前遍历，找到第一个场景号比当前要插入的场景号小的那条记录，调用js原生splice方法插入。复杂度(n)
successUploadFile(response, file) {
  console.log('进入uploadFile');
  const reader = new FileReader();
  console.log('response---', response);
  console.log(file);
  if (response.status === 200) {
    reader.readAsText(file, 'UTF-8');
    reader.onload = (event) => {
      const jsonConfig = JSON.parse(event.target.result);
      console.log(jsonConfig);
      if (jsonConfig && jsonConfig.robotConfig && jsonConfig.robotConfig.sceneConfig) {
        const newSceneConfig = jsonConfig.robotConfig.sceneConfig[0];
        const len = this.tableDataOrigin.length;
        for (let i = len - 1; i >= 0; i--) {
          if (this.tableDataOrigin[i].id < newSceneConfig.id) {
            this.tableDataOrigin.splice(i + 1, 0, JSON.parse(JSON.stringify(newSceneConfig)));
            break;
          }
        }
        this.tableData = this.tableDataOrigin.slice((this.currentPage - 1) * 10, this.currentPage * 10);
        console.log('this.tableDataOrigin---', this.tableDataOrigin);
      } else {
        console.log('不存在场景配置字段');
        throw new Error('不存在场景配置字段');
      }
    };
  } else {
    console.log('后端处理上传的全局配置文件失败');
    throw new Error('后端处理上传的全局配置文件失败');
  }
}
